# GSD: Map Codebase

Analyze existing codebases for brownfield projects to understand current state before planning changes.

## Workflow Source

This workflow is defined in detail at `_gsd/workflows/map-codebase.md`.

**CRITICAL:** Load the FULL content from `@_gsd/workflows/map-codebase.md`, READ its entire contents, and follow its directions exactly!

## Quick Reference

This command will:
1. Scan directory structure and technology stack
2. Analyze architecture patterns and data flow
3. Identify pain points and technical debt
4. Generate CODEBASE.md with SWOT analysis
5. Update PROJECT.md to reflect current reality

## Usage

```
/gsd-map-codebase
```

Use this for existing codebases before planning enhancements or refactoring.
