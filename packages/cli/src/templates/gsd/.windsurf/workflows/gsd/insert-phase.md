# GSD: Insert Phase

⚠️ CAUTION: Inject urgent work between existing phases by renumbering subsequent phases.

## Workflow Source

This workflow is defined in detail at `_gsd/workflows/insert-phase.md`.

**CRITICAL:** Load the FULL content from `@_gsd/workflows/insert-phase.md`, READ its entire contents, and follow its directions exactly!

## Quick Reference

This command will:
1. Confirm insertion urgency with user
2. Design new phase specification
3. Backup current state
4. Rename files and renumber phases
5. Update all references in project files

## Usage

```
/gsd-insert-phase [position]
```

⚠️ Only use when work is truly urgent and can't wait. Prefer `/gsd-add-phase` otherwise.
